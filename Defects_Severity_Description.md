# Defects Severity Description

This section categorizes defects based on their severity levels and provides a detailed description of each level. The severity levels include Critical, High, Medium, and Low. For each severity level, the following details are included:

- Definition
- Impact on the system
- Examples of defects
- Priority for fixing

| Severity Level | Definition | Impact on System | Examples of Defects | Priority for Fixing |
|----------------|------------|------------------|---------------------|---------------------|
| Critical       | A defect that causes complete failure of the system or a major part of it. | System is unusable, causing a halt in operations. | System crash, data loss, security breach. | Immediate fix required. |
| High           | A defect that significantly impairs functionality but does not cause a complete system failure. | Major functionality is impacted, but workarounds are possible. | Major feature not working, performance issues. | High priority, fix as soon as possible. |
| Medium         | A defect that affects some functionality but does not have a severe impact on operations. | Minor functionality is impacted, but the system is still usable. | Minor feature not working, occasional errors. | Medium priority, fix in the next release cycle. |
| Low            | A defect that has minimal impact on functionality and does not affect system operations. | Negligible impact on system usability. | Cosmetic issues, minor UI glitches. | Low priority, fix when convenient. |
